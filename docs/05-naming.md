# 05 — Naming rules

> **Version:** 0.1.0 | **Updated:** 2026-06-16

One concept, one name, everywhere. These rules are binding for the whole
library — every PR is checked against them.

---

## Rules

| Rule                                        | Detail                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Full words, never abbreviations             | `second` not `sec`; `nanosecond` not `nano` / `nanos`; `year` / `month` / `day` not `y` / `m` / `d`; `value` not `v`; `other` not `o`; `timezone` not `tz`; `factor`, `duration`, `count`, … in long form everywhere.                                                                                                                                                                                                                                                                                                                                |
| Name = purpose                              | A getter that returns the year is `year()`. A method's name states exactly what it does or returns. No "smart" abbreviations that require a comment to decode.                                                                                                                                                                                                                                                                                                                                                                                       |
| Singular = a component; plural = a quantity | `nanosecond()` is the sub-second component (`0..<1e9`); `total_milliseconds()` and `days_since_epoch()` are total quantities.                                                                                                                                                                                                                                                                                                                                                                                                                        |
| One reference point: `epoch`                | `epoch` always means the Unix epoch, `1970-01-01 00:00:00 UTC`. Every "amount since a reference" follows one pattern: `<unit>_since_epoch` (`days_since_epoch`, `seconds_since_epoch`) or `<unit>_since_midnight` (`nanoseconds_since_midnight`).                                                                                                                                                                                                                                                                                                    |
| `now` = the current reading, everywhere     | `Clock.now()`, `Instant.now()`, `Now.utc_*()`, `ZonedDateTime.now(timezone)`. The same word, the same shape.                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| Operators are reserved for affine algebra   | `Duration ± Duration`, `Instant ± Duration`, `Instant − Instant`, `DateTime ± Duration`, `DateTime − DateTime`. Calendar stepping (months / years, end-of-month clamping) uses named methods (`plus_days`, `plus_span`).                                                                                                                                                                                                                                                                                                                             |
| One concept = one name across the library   | Never `seconds` in one place and `secs` / `subsec_nanos` in another. Never `Locale` here and `locale_code` there.                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| Filenames                                   | `<type_snake>.mojo` for the primary type's home (`date.mojo` → `Date`, `datetime.mojo` → `DateTime`, `offset_datetime.mojo` → `OffsetDateTime`, `zoned_datetime.mojo` → `ZonedDateTime`). **Exception:** a pair of shape-matched types in the same conceptual space may share a file under a kind-name plural (`enums.mojo` houses `Weekday` + `Month`; `partials.mojo` houses `YearMonth` + `MonthDay`). Splitting these would scatter the iteration loops that the rest of the library (`strftime`, `rfc2822`, `natural`, `rrule`) does over both. |
| Imports                                     | Explicit, at the top of the file. Never wildcards. Never `from X import *`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |

---

## Canonical lexicon

Reuse exactly. Adding a synonym is a naming-convention violation.

| Concept                             | Name                                                                                                                                             |
| ----------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| Whole seconds of a duration         | `total_seconds()`                                                                                                                                |
| Sub-second component                | `nanosecond()`                                                                                                                                   |
| Total in a unit                     | `total_<unit>()` (`total_milliseconds`, `total_microseconds`, ...)                                                                               |
| Factory from a quantity             | `from_<unit>(<unit>)` — `from_seconds(seconds)`, `from_hours(hours)`, ...                                                                        |
| Point's value                       | `since_epoch()` / `seconds_since_epoch()` / `milliseconds_since_epoch()`                                                                         |
| Date's integer                      | `days_since_epoch()`                                                                                                                             |
| Time's integer                      | `nanoseconds_since_midnight()`                                                                                                                   |
| Current reading                     | `now()` / `Now.utc_*()` / `ZonedDateTime.now(timezone)`                                                                                          |
| Calendar step (named, not operator) | `plus_days()` / `minus_days()` / `plus_span()`                                                                                                   |
| Enum label                          | `name() -> StaticString`                                                                                                                         |
| Month / weekday number              | `Month.number()` (1..12) / `Weekday.iso_number()` (1..7)                                                                                         |
| Validity check                      | `is_<x>()` — `is_zero`, `is_negative`, `is_leap_year`, `is_utc`                                                                                  |
| Wire-format parse                   | `parse(text) raises -> <Type>` (every format module: `Rfc3339`, `Rfc2822`, `Asn1Time`, `IsoDuration`, `Strftime`, `HttpDate`, `RecurrenceRule`)  |
| Wire-format render                  | `format(<args>) raises -> String`                                                                                                                |
| Parametric type parameter           | Full word: `Timezone[transition_count, type_count]`, `Instant[clock]` — never `Timezone[N, T]` in production code (it's shorthand in docs only). |
